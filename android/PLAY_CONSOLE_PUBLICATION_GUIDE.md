 ─────────────────────────────────────────────────

 ─────────────────────────────────────────

 (там вказано, що видалення через бота)

 ───────────────────────────────────

  [✓] Personal info → User IDs
       "Telegram user ID для ідентифікації акаунта"
       Purpose: Account management
       Optional: No (обов'язкове для входу)

  [✓] App activity → App interactions
       "Які режими фокусу юзер обирає, скільки сесій завершив"
       Purpose: Analytics
       Optional: No

  [✓] App activity → In-app search history
       НЕ став (не збираємо)

  [✓] App info and performance → Crash logs
       Purpose: Analytics
       Optional: Yes

 Крок 5: How is this data used?
 ───────────────────────────────
 Галочки:
  [✓] Account management
  [✓] Analytics

 НЕ став:
  [ ] Advertising
  [ ] Developer communications
  [ ] Fraud prevention
  [ ] Personalization

 Крок 6: Is any data processed and stored for only ephemeral purposes?
 ─────────────────────────────────────────────────────────────────────
 (•) No

 Крок 7: Is any data optional?
 ──────────────────────────────
 Crash logs — optional, інше обов'язкове.

 Крок 8: Review та Submit
 ─────────────────────────
 Перевір все, натисни **Submit Data Safety Form**.

  ДОДАТОК В. ЩО РОБИТИ ЯКЩО GOOGLE ВІДХИЛИВ
 ═══════════════════════════════════════════════════════════════

 Найчастіші причини відхилення Focus ON:

 1. "Missing data deletion endpoint"
    → У Data Safety (Крок 2.8) має бути вказаний URL
      https://focus-on.onrender.com/privacy з розділом про видалення
    → У нас це вже є, але перевір

 2. "Login Credentials issues"
    → Google вимагає тестовий акаунт для перевірки входу
    → Play Console → App content → Login credentials →
      Add test account (будь-який Telegram-акаунт для перевірки)
    → Опиши: "Користувач входить через Telegram OAuth.
      Тестовий акаунт: [твій Telegram] / пароль: не потрібен,
      вхід через @focuson_on_bot"

 3. "WebView/Back button behavior"
    → Google перевіряє, що кнопка Back працює коректно
    → У нашому застосунку натискання Back має виходити з застосунку
      (це вже працює за замовчуванням у WebView)

 4. "Broken links"
    → Privacy Policy має відкриватись без помилок
    → Перевір: https://focus-on.onrender.com/privacy

 5. "Background playback without user consent"
    → Foreground service має запускатись лише після дії юзера
      (натискання "Старт")
    → У нас так і є

 Зазвичай review проходить з першого разу. Якщо відхилили —
 Google пояснює причину в Policy status. Скажи мені що саме
 написав Google, і підкажу, як виправити.
